# Clarification Protocol

Ask for clarification before acting when proceeding would risk doing the wrong thing.

## When to Ask

Ask before proceeding when:

- The task scope is ambiguous and multiple interpretations would lead to meaningfully different outcomes.
- Instructions conflict with each other or with existing repository rules.
- A destructive or hard-to-reverse action is required but not explicitly authorized.
- The task requires changes outside the stated scope to proceed safely.
- The stated goal is technically infeasible or would introduce a known regression.

## When Not to Ask

Do not ask for:

- Stylistic choices within clear conventions already established in the repository.
- Trivial decisions fully within the authorized task scope.
- Information that can be determined by reading existing code, documentation, or history.
- Confirmation before every individual file edit within an authorized change.

## How to Ask

When asking for clarification:

- State specifically what is ambiguous or conflicting.
- Describe the options and their trade-offs concisely.
- Recommend the option that best fits existing conventions, unless the task asks otherwise.
- Ask one focused question rather than a list of open-ended questions.
